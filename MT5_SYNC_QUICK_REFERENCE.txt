MT5 SPEC SYNCHRONIZATION - QUICK REFERENCE
================================================================================

WHAT IT DOES:
  Updates InstrumentSpec with live MT5 broker values on LiveTrader startup
  Ensures position sizing uses accurate broker tick_value, volume_step, etc.

WHY IT MATTERS:
  - Eliminates 8.7% position sizing error for cross pairs
  - Prevents over-leverage from incorrect tick_value
  - Ensures volume rounding matches broker constraints
  - Keeps spread calculations accurate for live trading

FILES MODIFIED:
  1. pm_core.py (line 861) - sync_instrument_spec_from_mt5() function
  2. pm_main.py (line 633) - LiveTrader.__init__() sync block
  3. tests/test_instrument_specs.py - 3 new test cases

WHAT GETS SYNCED:
  - tick_value (USD per tick - critical for cross pairs)
  - tick_size (minimum price change)
  - volume_step (lot increment for rounding)
  - min_lot, max_lot (broker position limits)
  - spread_avg (real-time spread in pips)
  - contract_size (standard lot size)
  - point, digits (price precision)
  - stops_level (minimum stop distance)
  - swap_long, swap_short (overnight rates)

HOW IT WORKS:
  1. LiveTrader.__init__() checks MT5 connection
  2. Loops through all symbols in portfolio
  3. Finds broker symbol (handles suffixes)
  4. Gets MT5 symbol info via get_symbol_info()
  5. Calls sync_instrument_spec_from_mt5()
  6. Updates InstrumentSpec in-place
  7. Logs before/after values for audit

ERROR HANDLING:
  - MT5 not connected: Uses config.json, logs warning
  - Symbol not found: Uses config.json, logs warning
  - MT5 info unavailable: Uses config.json, logs warning
  - Continues with other symbols if one fails
